# ⚡ 전력 이슈 뉴스·특허 아카이브

반도체 클러스터·AI 데이터센터·**3대 메가프로젝트** 시대에 가장 큰 변수인 **전력 이슈** —
관련 **뉴스는 매일**, **특허는 매주** 자동 수집해 누적 아카이브하는 정적 웹사이트입니다.
사이트는 **📰 뉴스 / 📄 특허** 두 탭으로 구성됩니다.

- **API 키·결제 불필요** — 뉴스는 Google 뉴스 RSS, 특허는 Google Patents 공개 데이터.
- **완전 자동** — GitHub Actions가 수집 → 빌드 → GitHub Pages(`gh-pages`)로 배포합니다.
- **의존성 없음** — 순수 파이썬 표준 라이브러리로만 동작합니다.

> 본 사이트는 이슈 **아카이브용**이며 특정 투자·정책 판단을 권유하지 않습니다.
> 뉴스 제목·요약·링크는 각 언론사 원문, 특허는 Google Patents 원문으로 연결됩니다.

## 화면 (인터랙티브 리더)
`index.html` 한 장짜리 앱이 전체 아카이브를 담아 **브라우저에서 통합검색·다중필터·정렬**합니다.
데이터가 페이지에 인라인되어 백엔드 없이, `file://` 로컬 열람과 GitHub Pages 모두 동작합니다.
- **📰 뉴스 / 📄 특허 탭** 전환
- **통합 검색**: 제목·요약·출처·출원인·공개번호를 전 기간에서 즉시 검색
- **다중 필터**: 카테고리(다중), 특허 국가(🇰🇷/🇺🇸), 정렬(최신/오래된순), "✨ 새 항목"(지난 방문 이후)
- **가벼운 개요**: 누적/수집기간/최근/신규 스탯 타일 + 기간별 추이 스파크라인
- **특허 통계 뷰**(특허 탭 `목록 / 📊 통계` 토글): 출원인 랭킹(한/영 이름 정규화·병합) +
  **출원인 국적(추정) 타일 그리드 세계지도** + 공개청(KR/US) 분포 + 카테고리×공개청 히트맵.
  현재 필터(카테고리·국가·검색)를 그대로 반영. 국적은 출원인명 매핑(대기업)+한글명은 한국으로
  추정하며 '공개청'과는 다른 개념(미매핑은 '미상'). 지도는 지리 데이터 없이 자기완결형 타일 배치
- **공유 가능**: 필터·뷰 상태가 URL 해시에 반영 → 링크로 특정 필터/통계 화면 공유
- 라이트/다크 자동, 모바일 대응

## 카테고리
뉴스: ⚡전력수급·전력난 · 🔌송·변전·전력망 · ☢️원전·SMR · 🌿재생에너지 · 🖥️데이터센터·AI 전력 ·
🏗️반도체 클러스터·메가프로젝트 · 🏛️전기요금·정책·한전 · 🏭전력설비·산업
특허: 같은 8칸을 기술 키워드(전력변환·송전·변압기·에너지저장·SMR·전력반도체·스마트미터·차단기 등)로 튜닝.

## 구성
| 파일 | 역할 |
|------|------|
| `news_config.py` / `patent_config.py` | 카테고리·검색 키워드·경로 설정 (뉴스 / 특허) |
| `news_source.py` | Google 뉴스 RSS 수집·파싱 (차단/오프라인 시 MOCK 폴백) |
| `patent_source.py` | Google Patents 공개 JSON 수집 (실패/오프라인 시 MOCK 폴백) |
| `news_archive.py` / `patent_archive.py` | 날짜별(뉴스)·주별(특허) JSON 누적 + 중복 제거 |
| `site_render.py` | 인터랙티브 SPA 렌더 (인라인 데이터 + 검색/필터/정렬/개요, 라이트·다크) |
| `build_site.py` | 수집→누적→사이트(index.html) 재생성 오케스트레이션 |
| `.github/workflows/daily-power-news.yml` | 매일 08:00 KST → 뉴스 수집 후 배포 |
| `.github/workflows/weekly-power-patents.yml` | 매주 월 08:20 KST → 특허 수집 후 배포 |

## 로컬 미리보기
```bash
python build_site.py                 # 뉴스+특허 둘 다 수집 후 site/ 생성 (의존성 없음)
# site/index.html 을 브라우저로 그냥 열기 (데이터 인라인이라 file:// 에서도 검색·필터 동작)
# (네트워크 차단·오프라인이면 자동으로 샘플(MOCK) 데이터로 빌드됩니다)
```
개별 수집: `python build_site.py --collect news` / `--collect patents` / `--collect none`(재빌드만).

## GitHub Pages 공개 (최초 1회 설정)
1. 코드를 push 하고 **Actions 탭**에서 워크플로를 한 번 수동 실행(`Run workflow`) → `gh-pages` 생성.
2. **Settings → Pages → Source** 를 **Deploy from a branch → `gh-pages` / `(root)`** 로 지정.
3. `https://ahn0405-cpu.github.io/power-news-patents-archive/` 에서 열립니다.
   이후 뉴스는 매일, 특허는 매주 자동 갱신됩니다.

> 두 워크플로는 실행마다 `gh-pages`에서 **뉴스·특허 아카이브를 모두 복원**한 뒤 자기 담당분만
> 더해 사이트 전체를 다시 만듭니다 → 어느 쪽이 돌아도 다른 탭 데이터는 유지, 히스토리는 항상 1커밋.
> 두 워크플로는 같은 concurrency 그룹이라 `gh-pages` 배포가 서로 겹치지 않습니다.

## 특허 소스에 대한 참고 (중요)
특허는 뉴스와 달리 완전 무료·무키인 공개 RSS가 없어, **Google Patents의 공개 JSON 엔드포인트**를
사용합니다. **무키로 한국·미국을 한 번에** 수집할 수 있는 대신 **비공식**이라, 구글이 차단하면
그 주 특허 수집이 실패할 수 있습니다. 이 경우 **특허 탭만 비고 뉴스 탭은 정상 동작**합니다
(MOCK 폴백/그레이스풀 실패). 안정성이 더 필요하면 한국은 무료 **KIPRIS/data.go.kr 키**로 승격하는
방식으로 `patent_source.py` 의 수집 함수만 교체하면 됩니다(나머지 누적·렌더는 그대로).

## 시각·키워드 편집
- 수집 시각: 각 워크플로의 `cron`(UTC, KST−9). 예: 08:00 KST → `0 23 * * *`.
- 키워드/카테고리: `news_config.py` / `patent_config.py` 의 `CATEGORIES`.
- 건수·주기 등 세부: `.env`(또는 워크플로 env)의 `NEWS_*` / `PATENT_*`.
